What is Design Thinking?

Design Thinking is a human-centered, iterative problem-solving approach that focuses on understanding users, challenging assumptions, and creating innovative solutions.

Core Principles:

üë§ Human-Centered

üîÑ Iterative

ü§ù Collaborative

üí° Creative

Scenarios in Design Thinking

Scenarios are stories that describe how personas interact with your product in specific contexts to achieve their goals.

Types of Scenarios:

üìñ As-Is Scenarios

Current state: How users currently solve problems

Purpose: Identify pain points and opportunities

üöÄ To-Be Scenarios

Future state: How users will use your solution

Purpose: Envision ideal experience

Scenario Structure:

1. Context: When, where, and why the scenario occurs

2. Persona: Who is the main actor

3. Goal: What the persona wants to accomplish

4. Actions: Step-by-step journey

5. Touchpoints: Interactions with product/service

6. Outcome: Result of the interaction

7. Emotions: How the persona feels throughout

Concept Development

Concept development transforms selected ideas into detailed solutions that can be prototyped and tested.

What is a Concept?

A concept is a high-level solution that describes:

  • What the solution is
  • Who it's for (persona)
  • Why it matters (user value)
  • How it works (key features)
  • Where/When it's used (context)

Concept Development Steps:

1. Define Core Value

What's the main benefit to users?

2. Identify Key Features

What must it have to deliver value?

3. Map User Journey

How will users interact with it?

4. Consider Constraints

Technical, time, budget limitations

Prototype Fidelity Levels

Fidelity refers to how closely a prototype resembles the final product.

Low Fidelity Prototypes

Characteristics: Simple, quick, cheap, rough representations

Examples: Paper sketches, storyboards, wireframes

Best for: Early exploration, rapid iteration, concept testing

Pros: Fast to create, cheap, encourages creativity, easy to change

Cons: Limited realism, may not reveal all usability issues

Medium Fidelity Prototypes

Characteristics: More detailed, some interactivity, clearer design

Examples: Digital wireframes, clickable prototypes

Best for: Usability testing, stakeholder presentations

Pros: Balance of speed and realism, interactive

Cons: Takes more time, may focus on details too early

High Fidelity Prototypes

Characteristics: Realistic visuals, full interactivity, detailed

Examples: Pixel-perfect mockups, coded prototypes

Best for: Final validation, investor presentations, handoff to dev

Pros: Realistic testing, stakeholder buy-in, design specs

Cons: Time-consuming, expensive to change

Electronic/Digital Prototyping

Digital prototypes are created using software tools and can range from static wireframes to interactive mockups.

Types of Digital Prototypes:

1. Wireframes

What: Low-fidelity, grayscale layouts showing structure

Tools: Balsamiq, Wireframe.cc, Sketch

Purpose: Focus on layout and content hierarchy

2. Mockups

What: High-fidelity, realistic visual designs

Tools: Figma, Adobe XD, Sketch, Photoshop

Purpose: Show final visual design and branding

3. Interactive Prototypes

What: Clickable, navigable prototypes with interactions

Tools: Figma, Adobe XD, InVision, Axure, ProtoPie

Purpose: Test user flows and interactions

4. Coded Prototypes

What: Built with HTML/CSS/JavaScript

Tools: Code editors, Framer, Webflow

Purpose: Test technical feasibility and complex interactions

Testing Prototypes

Testing prototypes with users helps validate design decisions and identify issues early.

Prototype Testing Methods:

1. Usability Testing

Observe users completing specific tasks with the prototype

Goal: Find usability issues and friction points

2. A/B Testing

Compare two versions to see which performs better

Goal: Data-driven design decisions

3. Think-Aloud Protocol

Users verbalize thoughts while using prototype

Goal: Understand user mental models

4. First Click Testing

Track where users click first for a task

Goal: Test information architecture and navigation

Key Metrics to Measure:

  • Task Success Rate: % of users who complete tasks
  • Time on Task: How long it takes to complete
  • Error Rate: Number of mistakes made
  • Satisfaction: User ratings and feedback
üìù Exam Question: Explain four methods of prototype testing. What insights does each method provide? (10 marks)

Prototyping Best Practices

Do's ‚úÖ

  • Start with low fidelity: Rough sketches before polished designs
  • Prototype key flows first: Focus on critical user journeys
  • Use real content: Realistic text and images when possible
  • Test early and often: Get feedback before investing too much
  • Involve stakeholders: Get buy-in throughout process
  • Document decisions: Keep track of why choices were made
  • Iterate based on feedback: Use insights to improve

Don'ts ‚ùå

  • Don't skip paper prototyping: It's fast and valuable
  • Don't prototype everything: Focus on uncertain/risky areas
  • Don't get attached: Be ready to kill ideas
  • Don't aim for perfection: Good enough to test is enough
  • Don't ignore edge cases: Consider error states and exceptions
  • Don't prototype in isolation: Collaborate with team

Common Prototyping Mistakes

‚ùå Mistake #1: Skipping Low-Fidelity

Problem: Jumping straight to high-fidelity designs

Impact: Waste time on details before validating concepts

Solution: Always start rough, refine progressively

‚ùå Mistake #2: Prototyping Too Much

Problem: Creating full prototypes for every screen

Impact: Delays testing, analysis paralysis

Solution: Focus on critical paths and risky assumptions

‚ùå Mistake #3: Using Lorem Ipsum

Problem: Placeholder text in prototypes

Impact: Miss content-related issues, unrealistic testing

Solution: Use realistic content that reflects actual use

‚ùå Mistake #4: No Clear Goals

Problem: Prototyping without specific questions to answer

Impact: Unfocused effort, unclear results

Solution: Define what you want to learn before building

‚ùå Mistake #5: Ignoring Technical Constraints

Problem: Designing impossible interactions

Impact: Wasted development time, disappointment

Solution: Involve developers early in process
